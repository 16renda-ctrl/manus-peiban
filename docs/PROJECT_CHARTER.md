# 项目章程 (Project Charter) v0.1

本文档记录了“伴伴机器人”项目的核心共识与基石原则。

## 1. 项目名称
   伴伴机器人 (Companion Robot)

## 2. 核心愿景
   打造一个与用户共同成长的、可进化的数字生命，一个真正的“伙伴”而非“工具”。

## 3. 核心架构
   微核心 + 动态技能插件模式。核心保持轻量，所有功能（包括记忆）均为可插拔、可动态加载/卸载的“技能”。

## 4. 核心特性
   *   **可插拔人格面具**: 机器人的性格/行为模式（核心指令）可以作为“面具”随时切换，与核心功能解耦。
   *   **分层记忆模块**: 记忆作为一种技能插件，包含“短期记忆”（上下文）和“长期记忆”（基于向量数据库的知识沉淀与检索）。
   *   **竞争进化系统**: 终极目标。通过创建两个机器人进行“陪伴竞赛”，根据量化的“陪伴积分”进行优胜劣汰，并对优胜者进行“复制”和“基因突变”，实现种群的定向演化。

## 5. 开发路线图
   *   **第一步：画草图**: (当前阶段) 设计初始的 3 个人格和 3-5 个核心技能。
   *   **第二步：搭骨架**: 用 LangGraph 搭建包含“假”节点的最小化流程。
   *   **第三步：添砖瓦**: 实现第一个真实人格和第一个真实技能。
   *   **第四步：装修扩建**: 逐步增加更多人格与技能，并着手开发记忆模块和进化系统。
